# Task: `asr-timestamp` — 字符级时间戳 ASR

每个字带精确开始/结束时间戳，用 `;` 分隔。适合做对齐、字幕生成、配音节奏分析。

## System Prompt

```
你是一个多语种语音识别专家，能够理解捕捉在语音识别过程中的时序关系。你必须按着用户给定的模板进行输出，避免其他无关的输出内容。
```

## User Prompt

```
请转录这段音频文件。对于识别出的每一个字请提供其精确的开始时间和结束时间。
你需要按着一字一行的格式来排列结果，每一行用';'隔开。每一行的由三部分组成，分别为开始时间、结束时间、转写字符，并且用'-'将它们分割开。要注意开始时间和结束时间的单位为秒，可以精确到小数点后两位。
可以参考下面的模板：
{开始时间}-{结束时间}-{转写字符};{开始时间}-{结束时间}-{转写字符};...{开始时间}-{结束时间}-{转写字符};
注意你只能按着模板输出结果，请勿输出其它无关的信息和内容。
```

## 示例输出

```
1.23-1.33-于;1.33-1.47-小;1.47-1.77-惠;2.33-2.42-你;2.42-2.55-把;2.55-2.70-冰;...
```

## 解析

模板严格 `start-end-char;`，单位秒、保留两位小数。简单解析：

```ts
result.split(/;\s*/).filter(Boolean).map(line => {
  const [start, end, ...rest] = line.split("-");
  return { start: Number(start), end: Number(end), char: rest.join("-") };
});
```

> 注意：字符本身可能含 `-`（很少见），所以用 `rest.join("-")` 兜底。

## 多段拼接

多段并发时，每段时间戳是该段相对零点。重组成全局时间戳时，要给第 N 段加上 `segments[N].start_sec` 偏移；JSON 里 `segments[].start_sec` 已经写好了。

## CLI

```bash
bun run {baseDir}/scripts/main.ts \
  --task asr-timestamp \
  --path ./meeting.wav \
  --out-dir ./out/timestamp
```
